# v0.2 后端与权限链路端到端冒烟清单

## 目标

用于在一次联调中快速确认以下链路是否正常：

- 平台管理端登录、节点切换、角色路由守卫生效
- admin 平台权限分层生效：`platform:admin`、`platform:user:manage`、`platform:manage_scope`
- web 业务端应用角色同步生效：`/api/v1/auth/app-roles`、`/api/v1/auth/app-permissions`
- 后端接口对排班与请假的应用权限收口生效
- 普通员工只可操作自己的请假记录，只可查看自己的已发布排班
- 无权限场景能够被 UI 与后端同时拦截

## 建议执行环境

- 后端服务启动：`go run ./cmd/server`
- admin 前端启动：`cd frontend/admin && pnpm dev`
- web 前端启动：`cd frontend/web && pnpm dev`
- 建议使用两个浏览器会话：
  - 会话 A：admin 平台端
  - 会话 B：web 业务端

## 最小测试数据准备

执行冒烟前，至少准备以下数据：

- 1 个平台管理员账号 `platform_admin`
- 1 个机构管理员账号 `org_admin`
- 1 个科室管理员账号 `dept_admin`
- 1 个普通业务用户账号，绑定员工 `employee_user`
- 该员工所在节点至少有 1 个部门节点，且当前账号可切换到该节点
- 至少 3 名员工，均绑定在同一业务节点：
  - 员工 A：分配 `app:schedule_admin`
  - 员工 B：分配 `app:scheduler`
  - 员工 C：分配 `app:leave_approver`
  - 员工 D：仅保留系统默认 `app:employee`
- 至少 1 个班次、1 条规则、1 个分组
- 至少 1 条已发布排班，包含员工 D 在未来 7 天内的班次
- 至少 1 条待审批请假记录，便于验证审批流

## 权限矩阵速查

### 平台角色

| 角色 | 预期能力 |
| --- | --- |
| `platform_admin` | 可访问 admin 全量菜单，包括机构、系统配置、订阅、审计、平台账号、应用权限 |
| `org_admin` | 不可访问平台级看板/机构/系统配置/订阅/审计；可访问平台账号与管理范围内业务配置 |
| `dept_admin` | 不可访问平台账号与平台级功能；可访问员工、分组、班次、规则、应用权限 |

### 应用角色

| 应用角色 | 预期业务权限 |
| --- | --- |
| `app:schedule_admin` | `schedule:create`、`schedule:execute`、`schedule:adjust`、`schedule:publish`、`schedule:view:all`、`leave:approve`、`leave:view:node` |
| `app:scheduler` | `schedule:create`、`schedule:execute`、`schedule:adjust`、`schedule:view:node` |
| `app:leave_approver` | `leave:approve`、`leave:view:node` |
| `app:employee` | `schedule:view:self`、`leave:create:self`、`preference:edit:self` |

## A. 认证与节点链路

### A1. admin 登录成功

- [ ] 打开 `/admin/login`
- [ ] 使用 `platform_admin` 登录
- [ ] 预期：进入 `/admin/dashboard` 或符合角色的默认首页
- [ ] 预期：`/api/v1/auth/login` 返回 200，响应中带 `current_node`、`available_nodes`

### A2. web 登录成功并同步应用权限

- [ ] 打开 `/login`
- [ ] 使用 `employee_user` 登录
- [ ] 预期：若未选择节点，则进入 `/select-node`
- [ ] 选择绑定员工所在节点
- [ ] 预期：`/api/v1/auth/me`、`/api/v1/auth/app-roles`、`/api/v1/auth/app-permissions` 均返回 200
- [ ] 预期：前端 store 中 `boundEmployeeId` 不为空

### A3. 强制改密链路

- [ ] 预期：admin 跳回 `/admin/login`
