# M07 — 排班展示与交互

> **所属阶段**: 阶段二（第 7 周）  
> **前置依赖**: M06（排班管道）、M10（前端骨架）  
> **交付物**: 甘特图视图 + 表格视图 + 排班创建向导 + 统计看板

---

## 一、目标

实现排班数据的可视化展示和交互编辑，包括甘特图拖拽、表格编辑、创建向导和统计看板。

---

## 二、页面结构

```
/scheduling
  /scheduling/create        # 创建排班（向导式 Stepper）
  /scheduling/calendar      # 排班日历（甘特图 + 表格视图切换）
  /scheduling/history       # 排班历史列表
  /scheduling/:id           # 排班详情（查看/编辑）
```

## 三、实现方案

### 3.1 甘特图视图

**技术选型**: 基于 Canvas/SVG 自研轻量组件（或 ECharts 自定义系列）

**功能**:

| 功能 | 实现方式 |
|------|---------|
| 纵轴：员工列表 | 虚拟滚动（支持 200+ 员工） |
| 横轴：日期时间线 | 周视图（7 天）/ 月视图（30 天）切换 |
| 班次色块 | 根据 shift.color 渲染，同日多班次垂直堆叠 |
| 双击添加 | 双击空白区域 → 弹出班次选择 → 调用调整 Pipeline |
| 拖拽调整 | 拖动色块到其他日期/员工 → 调用调整 Pipeline |
| 键盘删除 | 选中色块 + Delete → 确认后删除 |
| 冲突高亮 | violations 对应的色块显示红色边框/图标 |
| 来源标记 | hover 色块显示排班来源（fixed/rule/fill/ai/manual） |

**性能要求**: 200 员工 × 30 天渲染 < 1s

### 3.2 表格视图

**布局**: 员工 × 日期矩阵

```
          | 3/23(一) | 3/24(二) | 3/25(三) | ...
张三       |  白班    |  夜班    |  休      | ...
李四       |  夜班    |  休      |  白班    | ...
王五       |  白班    |  白班    |  夜班    | ...
```

**功能**:

- 单元格内显示班次名称（可配色）
- 点击单元格弹出编辑菜单（更换班次/删除）
- 支持批量选择 → 批量操作
- 行尾显示统计（本周白班数/夜班数/...）

### 3.3 排班创建向导

**Stepper 步骤**:

```
Step 1: 基本信息
  - 排班名称
  - 起止日期（日期范围选择器）

Step 2: 选择班次
  - 展示当前节点可用班次列表（多选）
  - 显示班次依赖关系图

Step 3: 配置人数需求
  - 每个选中班次 × 每天 → 输入需求人数
  - 支持「每日相同」快捷填充

Step 4: 个人偏好（可选）
  - 展示已提交的个人偏好列表
  - 管理员可添加临时调整

Step 5: 确认并生成
  - 汇总展示配置
  - 选择 Pipeline 类型（确定性 / AI 辅助）
  - 点击「生成排班」→ 等待 WebSocket 进度
  - 完成后跳转到排班详情页（甘特图预览）
```

### 3.4 统计看板

| 图表 | 类型 | 数据 |
|------|------|------|
| 工作量分布 | 柱状图 | 每位员工的排班次数（按班次类型堆叠） |
| 班次覆盖率 | 环形图 | 需求总人次 vs 已排人次 |
| 规则合规率 | 进度条 | 通过规则数 / 总规则数 |
| 加班/欠班 | 表格 | 超出/低于标准排班次数的员工列表 |

技术: ECharts

### 3.5 WebSocket 集成

```typescript
// 前端 WebSocket 监听
const ws = new WebSocket(`ws://api/v1/ws?token=${accessToken}`)

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data)
  switch (msg.type) {
    case 'schedule_progress':
      // 更新进度条
      progressStore.update(msg.schedule_id, msg.progress, msg.message)
      break
    case 'schedule_complete':
      // 刷新排班数据
      scheduleStore.refresh(msg.schedule_id)
      break
  }
}
```
