## 三、数据库设计

### 3.1 employees

```sql
CREATE TABLE employees (
    id            VARCHAR(64) PRIMARY KEY,
    org_node_id   VARCHAR(64) NOT NULL,
    name          VARCHAR(64) NOT NULL,
    employee_no   VARCHAR(32) DEFAULT NULL,
    phone         VARCHAR(20) DEFAULT NULL,
    email         VARCHAR(128) DEFAULT NULL,
    position      VARCHAR(64) DEFAULT NULL,         -- 职位（住院医/主治医/...）
    category      VARCHAR(32) DEFAULT NULL,          -- 分类（一线/二线/三线）
    status        ENUM('active', 'inactive') NOT NULL DEFAULT 'active',
    hire_date     DATE DEFAULT NULL,
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_org_node (org_node_id),
    INDEX idx_status (org_node_id, status),
    UNIQUE KEY uk_org_no (org_node_id, employee_no)
);
```

### 3.2 shifts

```sql
CREATE TABLE shifts (
    id            VARCHAR(64) PRIMARY KEY,
    org_node_id   VARCHAR(64) NOT NULL,
    name          VARCHAR(64) NOT NULL,
    code          VARCHAR(16) NOT NULL,             -- 班次编码（如 D/N/E）
    start_time    TIME NOT NULL,
    end_time      TIME NOT NULL,
    duration      INT NOT NULL,                      -- 时长（分钟）
    is_cross_day  BOOLEAN NOT NULL DEFAULT FALSE,
    color         VARCHAR(16) DEFAULT '#409EFF',
    priority      INT NOT NULL DEFAULT 0,            -- 排班执行优先级
    status        ENUM('active', 'disabled') NOT NULL DEFAULT 'active',
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_org_node (org_node_id),
    UNIQUE KEY uk_org_code (org_node_id, code)
);
```

### 3.3 shift_dependencies

```sql
CREATE TABLE shift_dependencies (
    id              VARCHAR(64) PRIMARY KEY,
    org_node_id     VARCHAR(64) NOT NULL,
    shift_id        VARCHAR(64) NOT NULL,            -- 源班次
    depends_on_id   VARCHAR(64) NOT NULL,            -- 依赖班次
    dependency_type ENUM('source', 'order') NOT NULL,-- 人员来源 / 执行顺序
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (shift_id) REFERENCES shifts(id),
    FOREIGN KEY (depends_on_id) REFERENCES shifts(id),
    INDEX idx_org_node (org_node_id),
    UNIQUE KEY uk_dep (shift_id, depends_on_id, dependency_type)
);
```

### 3.4 employee_groups + group_members

```sql
CREATE TABLE employee_groups (
    id            VARCHAR(64) PRIMARY KEY,
    org_node_id   VARCHAR(64) NOT NULL,
    name          VARCHAR(64) NOT NULL,
    description   VARCHAR(256) DEFAULT NULL,
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_org_node (org_node_id)
);

CREATE TABLE group_members (
    id            VARCHAR(64) PRIMARY KEY,
    group_id      VARCHAR(64) NOT NULL,
    employee_id   VARCHAR(64) NOT NULL,
    org_node_id   VARCHAR(64) NOT NULL,
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (group_id) REFERENCES employee_groups(id),
    FOREIGN KEY (employee_id) REFERENCES employees(id),
    INDEX idx_org_node (org_node_id),
    UNIQUE KEY uk_group_emp (group_id, employee_id)
);
```

### 3.5 leaves

```sql
CREATE TABLE leaves (
    id            VARCHAR(64) PRIMARY KEY,
    org_node_id   VARCHAR(64) NOT NULL,
    employee_id   VARCHAR(64) NOT NULL,
    leave_type    VARCHAR(32) NOT NULL,              -- annual/sick/personal/...
    start_date    DATE NOT NULL,
    end_date      DATE NOT NULL,
    reason        VARCHAR(256) DEFAULT NULL,
    status        ENUM('pending', 'approved', 'rejected') NOT NULL DEFAULT 'pending',
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    FOREIGN KEY (employee_id) REFERENCES employees(id),
    INDEX idx_org_node (org_node_id),
    INDEX idx_emp_date (employee_id, start_date, end_date)
);
```

## 五、关键业务逻辑

### 5.1 员工批量导入

- 接受 Excel 文件（`.xlsx`）
- 解析 → 校验（必填字段、工号唯一性）→ 批量插入
- 返回导入结果（成功条数 + 失败条数 + 失败原因）
- 使用 `excelize` 库

- 创建/审批请假后，标记该员工在请假日期范围内为「不可用」

## 六、验收标准

| 验收项 | 通过条件 |
|--------|---------|
| CRUD 全流程 | 员工/班次/分组/请假的增删改查 API 全部通过 |
