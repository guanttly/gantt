# M03 — 认证与权限

> **所属阶段**: 阶段一（第 2~3 周）  
> **前置依赖**: M01（项目骨架）、M02（组织树）  
> **交付物**: JWT 认证 + RBAC 权限 + 组织节点切换 + 路由中间件

---

## 一、目标

实现内建的认证体系，替代外部 `auth-server` 依赖。用户注册/登录后获取 JWT，Token 中携带当前组织节点信息，支持在多个关联节点间切换。

## 二、数据库设计

### 2.1 users 表

```sql
CREATE TABLE users (
    id          VARCHAR(64) PRIMARY KEY,
    username    VARCHAR(64) NOT NULL UNIQUE,
    email       VARCHAR(128) NOT NULL UNIQUE,
    phone       VARCHAR(20) DEFAULT NULL,
    password_hash VARCHAR(256) NOT NULL,
    status      ENUM('active', 'disabled') NOT NULL DEFAULT 'active',
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_email (email),
    INDEX idx_phone (phone)
);
```

### 2.2 roles 表

```sql
CREATE TABLE roles (
    id          VARCHAR(64) PRIMARY KEY,
    name        VARCHAR(64) NOT NULL UNIQUE,       -- platform_admin / org_admin / dept_admin / scheduler / employee
    display_name VARCHAR(64) NOT NULL,
    permissions JSON NOT NULL,                      -- ["schedule:create", "employee:read", ...]
    is_system   BOOLEAN NOT NULL DEFAULT FALSE,     -- 系统预置角色不可删除
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### 2.3 user_node_roles 表

```sql
CREATE TABLE user_node_roles (
    id          VARCHAR(64) PRIMARY KEY,
    user_id     VARCHAR(64) NOT NULL,
    org_node_id VARCHAR(64) NOT NULL,
    role_id     VARCHAR(64) NOT NULL,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (org_node_id) REFERENCES org_nodes(id),
    FOREIGN KEY (role_id) REFERENCES roles(id),
    UNIQUE KEY uk_user_node_role (user_id, org_node_id, role_id),
    INDEX idx_user (user_id),
    INDEX idx_node (org_node_id)
);
```

### 3.2 JWT 设计

**Token 载荷（Claims）**:

```go
type Claims struct {
    jwt.RegisteredClaims
    UserID      string `json:"uid"`
    OrgNodeID   string `json:"nid"`    // 当前登录的组织节点
    OrgNodePath string `json:"npath"`  // 当前节点的物化路径
    RoleName    string `json:"role"`   // 当前节点下的角色
}
```

**Token 生命周期**:

| Token 类型 | 有效期 | 用途 |
|-----------|--------|------|
| Access Token | 2 小时 | API 请求认证 |
| Refresh Token | 7 天 | 刷新 Access Token |

**签发流程**:

```
登录请求（username + password + org_node_id）
  → 校验密码
  → 查询 user_node_roles 确认用户有该节点权限
  → 签发 Access Token + Refresh Token
  → 返回 { access_token, refresh_token, expires_in, node_info }
```

### 3.3 组织节点切换

用户可关联多个组织节点。切换节点时，用旧 Token + 目标 node_id 签发新 Token：

```
POST /api/v1/auth/switch-node
Body: { "org_node_id": "node-xxx" }
Header: Authorization: Bearer <current_token>

→ 验证旧 Token 有效
→ 查询 user_node_roles 确认用户有目标节点权限
→ 签发新 Access Token（nid/npath/role 更新为目标节点）
→ 返回新 Token
```

### 3.4 RBAC 权限

**预置角色**:

| 角色 | 权限 |
|------|------|
| `platform_admin` | 全部权限 |
| `org_admin` | 组织管理 + 下级所有权限 |
| `dept_admin` | 科室管理 + 排班管理 + 人员管理 |
| `scheduler` | 排班创建/编辑 + 查看人员 |
| `employee` | 查看个人排班 + 提交偏好/请假 |

**权限定义**（细粒度）:

```go
var Permissions = map[string][]string{
    "platform_admin": {"*"},
    "org_admin":      {"org:*", "employee:*", "shift:*", "rule:*", "schedule:*", "leave:*", "ai:*"},
    "dept_admin":     {"employee:*", "shift:*", "rule:*", "schedule:*", "leave:*"},
    "scheduler":      {"employee:read", "shift:read", "rule:read", "schedule:*", "leave:read"},
    "employee":       {"schedule:read:self", "leave:create:self", "preference:*:self"},
}
```

**权限中间件**:

```go
// middleware.go

func RequirePermission(permission string) func(next http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            claims := GetClaims(r.Context())
            if !HasPermission(claims.RoleName, permission) {
                http.Error(w, "forbidden", 403)
                return
            }
            next.ServeHTTP(w, r)
        })
    }
}

// 使用示例
r.With(RequirePermission("schedule:create")).Post("/schedules", handler.Create)
```

## 四、API

```
POST   /api/v1/auth/register            # 注册（首次需管理员邀请或开放注册）
POST   /api/v1/auth/login               # 登录（username + password + org_node_id）
POST   /api/v1/auth/refresh             # 刷新 Token
POST   /api/v1/auth/password/reset      # 密码重置
POST   /api/v1/auth/switch-node         # 切换组织节点
```

## 六、密码安全

- 使用 `bcrypt` 哈希存储密码（cost = 10）
- 密码强度校验：≥ 8 字符，含大小写 + 数字
- 登录失败限流：同一 IP / 账号 5 分钟内 ≤ 5 次，超出锁定 15 分钟（Redis 计数）

## 七、验收标准

| 验收项 | 通过条件 |
|--------|---------|
| 注册 | 注册成功，密码以 bcrypt 存储 |
| 登录 | 登录返回 JWT，Claims 包含 uid/nid/npath/role |
| Token 验证 | 无效/过期 Token 返回 401 |
| 刷新 Token | Refresh Token 可换取新 Access Token |
| 节点切换 | 切换到有权限的节点返回新 Token；无权限返回 403 |
| 权限检查 | `employee` 角色无法调用 `POST /schedules`，返回 403 |
| 登录限流 | 连续 5 次失败后被锁定 |
