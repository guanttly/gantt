# 固定人员配置功能测试指南

## 功能概述

固定人员配置功能允许用户为任何班次（常规/加班/备班）配置固定人员，支持三种配置模式：
- **按周重复**：每周固定的某几天（如每周一、三、五）
- **按月重复**：每月固定的某几天（如每月1号、15号、30号）
- **指定日期**：具体的某几个日期（如2025-01-01, 2025-01-05）

## 工作流集成测试

### 1. 创建排班并验证固定人员自动填充

**步骤**：
1. 为某个班次配置固定人员（如：张三，每周一、三、五）
2. 在聊天界面中发起排班请求：
   ```
   请帮我安排下周的排班
   ```
3. 工作流进入固定班次阶段

**预期结果**：
- 工作流消息显示：
  ```
  🔒 固定班次自动填充完成
  
  共处理 1 个班次，填充了 3 个固定排班槽位：
    • [班次名称]: 3 天有固定人员
  
  ✅ 这些固定人员已自动排入班表，不再参与AI排班。
  ```
- 用户确认后，这些固定人员的排班已经预填充
- AI排班阶段不会再为这些时段和人员分配班次

### 2. 验证按月重复的计算

**步骤**：
1. 配置李四每月1号、15号、30号上夜班
2. 发起跨月排班（如：2025-01-15 到 2025-02-15）

**预期结果**：
- 固定班次阶段计算出以下日期：
  - 2025-01-15
  - 2025-01-30
  - 2025-02-01
  - 2025-02-15
- 李四在这些日期上被自动分配夜班

### 3. 验证指定日期模式

**步骤**：
1. 配置王五在 2025-01-01, 2025-01-05 上白班
2. 发起包含这些日期的排班

**预期结果**：
- 王五在 2025-01-01 和 2025-01-05 被自动分配白班
- 其他日期不受影响

### 4. 验证生效时间范围

**步骤**：
1. 配置赵六每周一上班，生效时间：2025-01-01 到 2025-01-31
2. 发起跨越生效期的排班（如：2024-12-25 到 2025-02-05）

**预期结果**：
- 只有 2025-01-01 到 2025-01-31 期间的周一被自动分配
- 2024-12-30（周一）不分配（生效时间之前）
- 2025-02-03（周一）不分配（生效时间之后）

## 常见问题排查

### 问题1：固定人员列显示"-"但实际已配置

**排查步骤**：
1. 打开浏览器开发者工具 - Network
2. 查看 `/shifts/{shiftId}/fixed-assignments` 的响应
3. 确认API返回了配置数据
4. 检查前端 `fetchFixedAssignmentStatus` 函数是否正确处理响应

### 问题2：保存后刷新列表，配置消失

**排查步骤**：
1. 检查 `batchCreateFixedAssignments` API的响应
2. 确认后端是否成功保存到数据库
3. 查询数据库：
   ```sql
   SELECT * FROM shift_fixed_assignments 
   WHERE shift_id = 'xxx' AND deleted_at IS NULL;
   ```

### 问题3：工作流未自动填充固定人员

**排查步骤**：
1. 检查工作流日志，确认进入了固定班次阶段
2. 查看 `CalculateFixedSchedule` 函数的返回值
3. 确认日期范围是否包含固定人员的生效日期
4. 检查 `isEffectiveOn` 函数的逻辑
