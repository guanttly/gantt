# V3排班上下文架构重构方案

**文档版本**: v1.0  
**创建日期**: 2026-01-28  
**目标**: 建立清晰的递归3层上下文架构

---

## 一、核心问题

### 当前架构的根本缺陷

```go
// ❌ 错误的设计：任务包含多个班次，但候选人员是任务级别的
CoreV3TaskContext {
    Task           *ProgressiveTask  // 可能包含多个班次
    CandidateStaff []*Employee       // ❌ 任务级，但不同班次候选人员不同！
    Shifts         []*Shift          // 多个班次
    OccupiedSlots  map[string]map... // ❌ 混杂所有班次的占位
}

// 问题：
// 1. 候选人员应该是"班次级别"的，而不是"任务级别"
// 2. 一个员工在某班次已占位，但可能是另一班次的候选人
// 3. 数据全部混在一起，没有层次，调用混乱
```

### 业务真相

```
现实情况：
- 任务1: 安排【早班】和【中班】，2026-02-01 ~ 2026-02-03
  - 执行早班时：张三已被中班占位 2月1日，不能是早班候选人
  - 执行中班时：张三可以是中班候选人（不冲突）
  
当前代码：
- CandidateStaff 在任务级别就确定了
- 导致张三要么全部可选，要么全部不可选
- 无法做到"按班次动态过滤"
```

## 二、正确的架构：递归3层结构

### 2.1 层次关系

```
L1: CreateV3Context                    (总工作流上下文)
    └─ 管理：全局数据（所有员工、所有班次、总草稿）
        │
        ▼ 执行任务
    L2: TaskExecutionContext           (任务执行上下文)
        └─ 管理：任务涉及的班次、任务范围的占位
            │
            ▼ 拆分成单班次
        L3: ShiftTaskContext           (单班次任务上下文) ★
            └─ 管理：单个班次、该班次的候选人员（动态计算）
                │
                ▼ 执行LLM
            ShiftScheduleDraft         (单班次结果)
                │
                ▼ 递归合并
            L3 → L2 → L1
```

### 2.2 关键设计原则

| 原则 | 说明 |
|------|------|
| **每层管理自己的数据** | L1管全局、L2管任务、L3管单班次 |
| **候选人员动态计算** | 在L3执行前，根据当时的占位状态计算 |
| **递归合并** | L3结果→L2→L1，逐层更新占位和草稿 |
| **数据继承清晰** | 只读数据向下传递，可写数据向上合并 |

### 3.2 L2: TaskExecutionContext（任务执行上下文）

```go
// 文件位置: agents/rostering/internal/workflow/schedule_v3/utils/task_context.go (新文件)

type TaskExecutionContext struct {
    // ========== 从L1继承的只读数据 ==========
    AllStaff      []*Employee
    Rules         []*Rule
    PersonalNeeds map[string][]*PersonalNeed

    // ========== 任务范围数据（从L1过滤/切片） ==========
    Task         *ProgressiveTask
    TargetShifts []*Shift                    // 该任务涉及的班次（可能多个）
    
    // 深拷贝L1的可写数据（任务执行时的快照）
    OccupiedSlots     []StaffOccupiedSlot
    StaffRequirements []ShiftDateRequirement   // 仅该任务相关
    FixedAssignments  []CtxFixedShiftAssignment // 仅该任务相关
    
    WorkingDraft *ScheduleDraft               // 引用L1的WorkingDraft

    // ========== 任务结果 ==========
    TaskResult *TaskResult
}

// SplitIntoShiftContexts 拆分成多个单班次上下文
func (tctx *TaskExecutionContext) SplitIntoShiftContexts() []*ShiftTaskContext {
    contexts := make([]*ShiftTaskContext, 0, len(tctx.TargetShifts))
    
    for _, shift := range tctx.TargetShifts {
        shiftCtx := &ShiftTaskContext{
            // 继承只读数据
            AllStaff:      tctx.AllStaff,
            PersonalNeeds: tctx.PersonalNeeds,
            
            // 班次特定
            Task:        tctx.Task,
            TargetShift: shift,
            
            // 相关班次（任务中的其他班次 + 规则关联的班次）
            RelatedShifts: buildRelatedShifts(shift, tctx.TargetShifts, tctx.Rules),
            
            // 过滤与该班次相关的规则
            FilteredRules: filterRulesByShift(tctx.Rules, shift.ID),
            
            // 深拷贝占位（避免并发修改）
            OccupiedSlots: deepCopy(tctx.OccupiedSlots),
            
            // 过滤该班次的数据
            StaffRequirements: filterByShift(tctx.StaffRequirements, shift.ID),
            FixedAssignments:  filterByShift(tctx.FixedAssignments, shift.ID),
            CurrentDraft:      extractShiftDraft(tctx.WorkingDraft, shift.ID),
        }
        
        // ★ 关键：动态计算该班次的候选人员
        shiftCtx.CandidateStaff = shiftCtx.ComputeCandidateStaff()
        
        // 预构建LLM缓存
        shiftCtx.BuildLLMCache()
        
        contexts = append(contexts, shiftCtx)
    }
}

// buildRelatedShifts 构建相关班次列表
func buildRelatedShifts(
    targetShift *Shift,
    taskShifts []*Shift,
    rules []*Rule,
) []*Shift {
    relatedMap := make(map[string]*Shift)
    
    // 1. 添加任务中的其他班次
    for _, shift := range taskShifts {
        if shift.ID != targetShift.ID {
            relatedMap[shift.ID] = shift
        }
    }
    
    // 2. 从规则中提取关联班次
    for _, rule := range rules {
        // 如果规则涉及当前班次
        if containsShiftID(rule.RelatedShiftIDs, targetShift.ID) {
            // 添加规则中的其他关联班次
            for _, relatedID := range rule.RelatedShiftIDs {
                if relatedID != targetShift.ID {
                    // 从全局班次列表中查找（需要传入或从上下文获取）
                    if shift := findShiftByID(taskShifts, relatedID); shift != nil {
                        relatedMap[relatedID] = shift
                    }
                }
            }
        }
    }
    
    // 转换为数组
    result := make([]*Shift, 0, len(relatedMap))
    for _, shift := range relatedMap {
        result = append(result, shift)
    }
    return result
}

// filterRulesByShift 过滤与班次相关的规则
func filterRulesByShift(rules []*Rule, shiftID string) []*Rule {
    result := make([]*Rule, 0)
    for _, rule := range rules {
        // 规则的 RelatedShiftIDs 包含当前班次
        if containsShiftID(rule.RelatedShiftIDs, shiftID) {
            result = append(result, rule)
        }
    }
    return result
    
    return contexts
}

// MergeShiftResults 合并所有单班次结果
func (tctx *TaskExecutionContext) MergeShiftResults(results []*ShiftTaskContext) {
    tctx.TaskResult = &TaskResult{
        TaskID:         tctx.Task.ID,
        ShiftSchedules: make(map[string]*ShiftScheduleDraft),
    }
    
    for _, r := range results {
        if r.Result != nil {
            tctx.TaskResult.ShiftSchedules[r.TargetShift.ID] = r.Result
            tctx.OccupiedSlots = mergeOccupiedSlots(tctx.OccupiedSlots, r.OccupiedSlots)
        }
    }
}
```

**职责**：
- 管理任务涉及的班次（可能多个）
- 拆分成多个 ShiftTaskContext
- 合并多个班次的结果
- 不负责候选人员计算（交给L3）

### 3.3 L3: ShiftTaskContext（单班次任务上下文）★ 核心

```go
// 文件位置: agents/rostering/internal/workflow/schedule_v3/utils/shift_task_context.go (新文件)

type ShiftTaskContext struct {
    // ========== 从L2继承的只读数据 ==========
    AllStaff      []*Employee
    PersonalNeeds map[string][]*PersonalNeed

    // ========== 单班次特定数据 ==========
    Task        *ProgressiveTask
    TargetShift *Shift                       // 当前处理的班次（单个）
    
    // RelatedShifts 相关班次（用于规则判断，如早晚班间隔）
    // 包含：1) 任务涉及的其他班次  2) 规则关联的班次
    RelatedShifts []*Shift
    
    // FilteredRules 过滤后的规则（仅与当前班次相关的）
    // 过滤规则：规则的 RelatedShiftIDs 包含 TargetShift.ID
    FilteredRules []*Rule
    
    // ★ 该班次的候选人员（动态计算）
    // 规则：AllStaff - 在目标日期已占位的 - 请假的
    CandidateStaff []*Employee
    
    OccupiedSlots     []StaffOccupiedSlot
    StaffRequirements []ShiftDateRequirement   // 仅该班次
    FixedAssignments  []CtxFixedShiftAssignment // 仅该班次
    CurrentDraft      *ShiftScheduleDraft

    // ========== LLM调用预构建数据 ==========
    StaffInfoForAI      []*StaffInfoForAI
    RequirementsMap     map[string]map[string]int
    FixedAssignmentsMap map[string][]string

    // ========== 执行结果 ==========
    Result *ShiftScheduleDraft
}

// ComputeCandidateStaff ★ 动态计算候选人员
func (sctx *ShiftTaskContext) ComputeCandidateStaff() []*Employee {
    targetDates := sctx.Task.TargetDates
    if len(targetDates) == 0 {
        return sctx.AllStaff
    }
    
    candidates := make([]*Employee, 0, len(sctx.AllStaff))
    
    for _, staff := range sctx.AllStaff {
        // 检查是否在目标日期已被占位
        occupied := false
        for _, date := range targetDates {
            if IsStaffOccupiedOnDate(sctx.OccupiedSlots, staff.ID, date) {
                occupied = true
                break
            }
        }
        
        if !occupied {
            // 检查是否请假
            if !hasLeaveOnDates(sctx.PersonalNeeds[staff.ID], targetDates) {
                candidates = append(candidates, staff)
            }
        }
    }
    
    return candidates
}

// BuildLLMCache 预构建LLM调用数据
func (sctx *ShiftTaskContext) BuildLLMCache() {
    // 1. 转换候选人员为AI格式
    sctx.StaffInfoForAI = NewStaffInfoListFromEmployees(sctx.CandidateStaff)
    
    // 2. 构建需求映射（LLM接口需要）
    sctx.RequirementsMap = ConvertRequirementsToMap(sctx.StaffRequirements)
    
    // 3. 构建固定排班映射
    sctx.FixedAssignmentsMap = ConvertFixedAssignmentsToMap(sctx.FixedAssignments)
}
```

**职责**：
- 管理单个班次的执行
- **动态计算候选人员**（核心！）
- **提供相关班次信息**（用于规则判断，如早晚班间隔）
- **过滤相关规则**（仅保留涉及当前班次的规则）
- 预构建LLM调用数据
- 直接调用LLM执行排班

**相关班次的必要性**：
```
场景：规则"早班和晚班之间需要间隔12小时"
- 当前执行早班排班时
- 需要知道晚班的时间信息（StartTime, EndTime）
- 需要检查员工是否在晚班有排班
- 因此 RelatedShifts 包含晚班信息
```

### 4.2 核心代码

```go
// ============================================================
// 文件位置: agents/rostering/internal/workflow/schedule_v3/core/actions.go
// ============================================================

func actExecuteTask(ctx context.Context, wctx engine.Context) error {
    sess := wctx.Session()
    logger := sess.Logger
    
    // 1. 获取 L1 上下文
    createCtx, err := GetCreateV3Context(ctx, wctx)
    if err != nil {
        return err
    }
    
    task := createCtx.ProgressiveTaskPlan.Tasks[createCtx.CurrentTaskIndex]
    
    // 2. 构建 L2 任务执行上下文
    taskCtx := BuildTaskExecutionContext(createCtx, task)
    
    // 3. 拆分成 L3 单班次上下文（动态计算候选人员）
    shiftContexts := taskCtx.SplitIntoShiftContexts()
    
    logger.Info("Split task into shift contexts",
        "taskID", task.ID,
        "shiftCount", len(shiftContexts))
    
    // 4. 执行每个班次
    for _, shiftCtx := range shiftContexts {
        logger.Info("Executing shift",
            "shiftID", shiftCtx.TargetShift.ID,
            "candidateCount", len(shiftCtx.CandidateStaff)) // ★ 动态计算的
        
        // 调用LLM（使用预构建的缓存）
        result, err := executeLLMForShift(ctx, shiftCtx)
        if err != nil {
            logger.Error("Failed to execute shift", "error", err)
            continue
        }
        
        shiftCtx.Result = result
    }
    
    // 5. 合并 L3 → L2
    taskCtx.MergeShiftResults(shiftContexts)
    
    // 6. 合并 L2 → L1
    MergeTaskResultToCreate(createCtx, taskCtx)
    
    // 7. 保存 L1
    return SaveCreateV3Context(ctx, wctx, createCtx)
}
```

// ============================================================

// StaffOccupiedSlot 人员占位（替代 map[string]map[string]string）
type StaffOccupiedSlot struct {
    StaffID   string `json:"staffId"`
    Date      string `json:"date"`
    ShiftID   string `json:"shiftId"`
    ShiftName string `json:"shiftName,omitempty"`
    Source    string `json:"source"` // fixed | draft | external
}

// ShiftDateRequirement 班次人员需求（替代 map[string]map[string]int）
type ShiftDateRequirement struct {
    ShiftID   string `json:"shiftId"`
    ShiftName string `json:"shiftName,omitempty"`
    Date      string `json:"date"`
    Count     int    `json:"count"`
}

// CtxFixedShiftAssignment 固定排班（替代 map[string]map[string][]string）
type CtxFixedShiftAssignment struct {
    ShiftID   string   `json:"shiftId"`
    ShiftName string   `json:"shiftName,omitempty"`
    Date      string   `json:"date"`
    StaffIDs  []string `json:"staffIds"`
}

// ============================================================
// 文件位置: agents/rostering/domain/model/context_helpers.go
// ============================================================

// IsStaffOccupiedOnDate 检查员工在指定日期是否已占位
func IsStaffOccupiedOnDate(slots []StaffOccupiedSlot, staffID, date string) bool {
    for _, slot := range slots {
        if slot.StaffID == staffID && slot.Date == date {
            return true
        }
    }
    return false
}

// FilterRequirementsByShift 过滤某班次的需求
func FilterRequirementsByShift(reqs []ShiftDateRequirement, shiftID string) []ShiftDateRequirement {
    result := make([]ShiftDateRequirement, 0)
    for _, req := range reqs {
        if req.ShiftID == shiftID {
            result = append(result, req)
        }
    }
    return result
}

// mergeOccupiedSlots 合并占位信息（去重追加）
func mergeOccupiedSlots(existing, updates []StaffOccupiedSlot) []StaffOccupiedSlot {
    result := make([]StaffOccupiedSlot, 0, len(existing)+len(updates))
    result = append(result, existing...)
    
    for _, newSlot := range updates {
        if !containsSlot(result, newSlot) {
            result = append(result, newSlot)
        }
    }
    
    return result
}

## 七、总结

### 核心改进

1. **候选人员动态计算**：在L3执行前，根据当前占位状态计算
2. **递归3层结构**：L1总览、L2任务、L3单班次，职责清晰
3. **强类型数组**：消除所有嵌套map
4. **预构建缓存**：L3构建时预转换LLM数据，避免重复

### 与旧架构对比

| 维度 | 旧架构 | 新架构 |
|------|--------|--------|
| 候选人员 | 任务级静态 | 班次级动态 |
