# V3排班单班次失败自动重试机制实现计划

## 一、需求概述

在V3排班链路中，当某个班次执行失败后，虽然AI指出了错误信息，但缺少自动重试逻辑。需要在单班次级别实现智能自动重试机制：
- 每个班次执行后立即校验
- 失败时清理本班次占位，调用AI分析失败原因
- 自动重试最多3次（每次带上历史失败信息）
- 部分成功时保存成功的班次，询问用户如何处理失败的班次
- 用户手动重试不限次数

## 二、设计原则

1. **重试位置**：在单班次执行循环内部（`executeAITask` 方法）
2. **校验时机**：单班次执行后立即校验，而非等所有班次执行完
3. **占位清理**：重试前清理本班次的占位记录，避免自我冲突
4. **历史保留**：清理占位但保留历史数据（简要语义化格式）供AI参考
5. **部分成功**：允许部分班次成功保存，失败班次由用户决策
6. **状态机扩展**：新增 `CoreV3StatePartialSuccess` 状态处理部分成功场景

## 三、技术架构

### 3.1 核心流程

```
任务开始 → 拆分成单班次子任务
    ↓
[遍历每个班次]
    ↓
[自动重试循环: 最多3次]
    ↓
    清理本班次占位 (occupiedSlots, shiftCurrentDraft)
    ↓
    执行单班次排班 (带重试上下文)
    ↓
    立即校验 (validateSingleShift)
    ↓
    校验通过？
        Yes → 记录成功，退出重试循环，继续下一班次
        No  → AI分析失败原因 (analyzeShiftFailureWithAI)
              ↓
              转为简要语义描述 ("安排了3人但需要5人")
              ↓
              retry < 3？
                  Yes → 记录到历史，继续重试
                  No  → 标记为失败，继续下一班次
    ↓
[所有班次处理完毕]
    ↓
    全部成功？
        Yes → 正常完成
        No  → 转到 CoreV3StatePartialSuccess 状态
              ↓
              展示失败详情，等待用户选择：
                  - 重试失败班次（不限次数）
                  - 跳过失败班次，保存成功部分
                  - 取消整个任务
```

### 3.3 新增方法

#### 1. validateSingleShift - 单班次即时校验
**位置**：`agents/rostering/internal/application/progressive_executor.go`

**功能**：
- 检查单个班次的人数是否满足要求
- 检查是否与 occupiedSlots 存在时间冲突
- 检查是否违反规则（针对该班次的规则）

**输入参数**：
- `shift *d_model.Shift` - 班次信息
- `draft *d_model.ShiftScheduleDraft` - 班次排班草案
- `requiredCount int` - 该班次每天需要的人数
- `staffList []*d_model.Employee` - 可用员工列表
- `rules []*d_model.Rule` - 规则列表
- `occupiedSlots map[string]map[string]string` - 已占用时间槽

**返回**：
- `*ValidationResult` - 校验结果（是否通过、问题列表）

#### 2. cleanShiftOccupiedSlots - 清理班次占位
**位置**：`agents/rostering/internal/application/progressive_executor.go`

**功能**：
- 从 occupiedSlots 中删除该班次的占位记录
- 从 shiftCurrentDraft 中删除该班次的排班数据
- 仅清理本班次相关的日期范围

**输入参数**：
- `shiftID string` - 班次ID
- `dateRange []string` - 日期列表
- `occupiedSlots map[string]map[string]string` - 占位映射
- `shiftCurrentDraft *d_model.ShiftScheduleDraft` - 当前草案

**返回**：无（直接修改入参）

#### 3. analyzeShiftFailureWithAI - AI分析失败原因
**位置**：`agents/rostering/internal/application/progressive_executor.go`

**功能**：
- 将历史失败信息转为简要语义化描述
- 调用AI分析失败根因
- 生成改进建议

**输入参数**：
- `ctx context.Context`
- `shift *d_model.Shift` - 班次信息
- `validationResult *ValidationResult` - 最新校验结果
- `historyFailures []string` - 历史失败描述列表
- `requiredCount int` - 需要的人数

**返回**：
- `failureDescription string` - 简要失败描述
- `recommendations string` - AI改进建议
- `error`

#### 4. executeAITask 改造 - 嵌入重试逻辑
**位置**：`agents/rostering/internal/application/progressive_executor.go` (L934-L1076)

**修改内容**：
- 在班次循环中嵌入重试循环（最多3次）
- 每次重试前清理占位
- 执行后立即校验
- 校验失败则调用AI分析并记录历史
- 所有班次处理完后，检查是否部分成功

#### 5. executeAITaskForSingleShift 增强 - 支持重试上下文
**位置**：`agents/rostering/internal/application/progressive_executor.go` (L1085-L1212)

**修改内容**：
- 增加 `retryContext *ShiftRetryContext` 参数
- 在构建AI提示词时注入重试上下文
- 提示词格式：
  ```

  这是第 {retryCount} 次重试
  
  历史失败原因：
  1. {historyFailure1}
  2. {historyFailure2}
  ```

## 四、状态机扩展

### 4.1 新增状态

**CoreV3StatePartialSuccess** - 部分成功状态

**说明**：当任务执行后，部分班次成功、部分班次失败（达到最大重试次数）时进入此状态

### 4.2 新增事件

1. **CoreV3EventRetryFailed** - 重试失败班次
   - 用户选择重试失败的班次
   - 不限重试次数

2. **CoreV3EventSkipFailed** - 跳过失败班次
   - 用户选择保存成功部分，跳过失败班次
   - 转到完成状态

3. **CoreV3EventCancelTask** - 取消任务
   - 用户选择取消整个任务
   - 不保存任何结果

### 4.3 状态转换定义

**位置**：`agents/rostering/internal/workflow/schedule_v3/core/definition.go`

```go
// 从校验结果转到部分成功
{
    From:  CoreV3StateValidateResult,
    Event: CoreV3EventValidated,
    To:    CoreV3StatePartialSuccess,  // 检测到部分成功时
    Act:   actOnPartialSuccess,
},

// 部分成功状态的转换
{
    From:  CoreV3StatePartialSuccess,
    Event: CoreV3EventRetryFailed,
    To:    CoreV3StatePartialSuccess,  // 重试后可能仍部分成功
    Act:   actRetryFailedShifts,
},
{
    From:  CoreV3StatePartialSuccess,
    Event: CoreV3EventSkipFailed,
    To:    CoreV3StateCompleted,
    Act:   actSkipFailedShifts,
},
{
    From:  CoreV3StatePartialSuccess,
    Event: CoreV3EventCancelTask,
    To:    CoreV3StateFailed,
    Act:   actCancelPartialTask,
},
```

### 4.4 Action 实现

**actOnPartialSuccess** - 处理部分成功场景

**位置**：`agents/rostering/internal/workflow/schedule_v3/core/actions.go`

**功能**：
- 检查 TaskResult.PartiallySucceeded 标志
- 构建失败班次展示消息
- 发送消息给用户，提供三个选项按钮
- 等待用户选择

**消息格式**：
```
⚠️ **部分班次执行失败**

成功完成：{successCount} 个班次
失败班次：{failedCount} 个

失败详情：
- **早班**（1月20-25日）：安排了3人但需要5人（已自动重试3次）
- **晚班**（1月22-24日）：存在时间冲突，张三和李四在同一时段已被其他班次占用（已自动重试3次）

请选择操作：
[重试失败班次] [跳过失败班次，保存成功部分] [取消整个任务]
```

**actRetryFailedShifts** - 重试失败班次

**功能**：
- 提取失败的班次列表
- 调用执行器重新执行这些班次（不限次数）
- 更新 TaskResult
- 如果全部成功，转到完成状态
- 如果仍有失败，停留在 CoreV3StatePartialSuccess，再次询问

**actSkipFailedShifts** - 跳过失败班次

**功能**：
- 保存成功的班次到 WorkingDraft
- 记录失败班次的信息到日志
- 转到完成状态

**actCancelPartialTask** - 取消部分成功任务

**功能**：
- 不保存任何结果
- 清理临时数据
- 转到失败状态

## 六、关键技术细节

### 6.1 占位清理逻辑

```go
func (e *ProgressiveTaskExecutor) cleanShiftOccupiedSlots(
    shiftID string,
    dateRange []string,
    occupiedSlots map[string]map[string]string,
    shiftCurrentDraft *d_model.ShiftScheduleDraft,
) {
    // 遍历每个日期
    for _, date := range dateRange {
        if daySlots, exists := occupiedSlots[date]; exists {
            // 遍历时间槽
            for timeSlot, slotShiftID := range daySlots {
                // 只清理本班次的占位
                if slotShiftID == shiftID {
                    delete(daySlots, timeSlot)
                }
            }
        }
    }
    
    // 清理草案中该班次的数据
    if shiftCurrentDraft != nil {
        delete(shiftCurrentDraft.ShiftAssignments, shiftID)
    }
}
```

### 6.2 语义化转换

```go
func formatFailureToSemantic(
    draft *d_model.ShiftScheduleDraft,
    validationResult *ValidationResult,
    requiredCount int,
) string {
    var issues []string
    
    // 人数问题
    if len(validationResult.StaffCountIssues) > 0 {
        assignedCount := len(draft.ShiftAssignments)
        issues = append(issues, fmt.Sprintf(
            "安排了%d人但需要%d人", assignedCount, requiredCount,
        ))
    }
    
    // 规则问题
    for _, issue := range validationResult.RuleComplianceIssues {
        issues = append(issues, issue.Message)
    }
    
    // 冲突问题
    for _, issue := range validationResult.ShiftRuleIssues {
        issues = append(issues, issue.Message)
    }
    
    return strings.Join(issues, "；")
}
```

### 7.2 集成测试

1. **自动重试流程测试**
   - 单班次失败1次后重试成功
   - 单班次失败3次全部失败

2. **部分成功状态测试**
   - 进入部分成功状态
   - 用户选择重试失败班次

3. **手动重试测试**
   - 手动重试失败班次成功
   - 手动重试多次仍失败

### 7.3 端到端测试场景

**场景1：人数不足自动重试成功**
- 第1次：AI安排了3人，需要5人 → 失败
- 第2次：AI根据反馈调整，安排了5人 → 成功

**场景2：规则冲突重试3次失败**
- 第1次：违反最大连续工作天数 → 失败
- 第2次：仍然违反相同规则 → 失败
- 第3次：仍然违反 → 最终失败，进入部分成功状态

**场景3：部分成功用户重试**
- 10个班次，8个成功，2个失败
- 进入部分成功状态，展示失败详情
- 用户选择重试
- 重试后1个成功，1个仍失败
- 再次进入部分成功状态
- 用户选择跳过，保存9个成功的班次

## 十、后续优化方向

1. **智能重试策略**
   - 根据失败类型选择不同的重试策略
   - 可重试错误 vs 不可重试错误自动识别

2. **并发班次执行**
   - 支持多个班次并发执行
   - 需要处理并发重试的复杂性

3. **重试历史持久化**
   - 将重试历史保存到数据库
   - 支持跨会话查看重试记录

4. **重试策略可配置**
   - 支持不同任务类型使用不同的重试策略
   - 支持自定义重试次数和延迟

5. **失败原因分类**
   - 自动分类失败原因（人数、冲突、规则等）
   - 针对不同类型提供不同的解决方案

## 十一、风险与注意事项

1. **AI分析性能**
   - 每次重试都调用AI分析可能增加延迟
   - 建议：失败分析使用轻量级prompt

2. **占位清理风险**
   - 清理错误可能导致数据不一致
   - 建议：清理前做快照，支持回滚

3. **无限重试风险**
   - 用户手动重试不限次数可能导致死循环
   - 建议：添加总体时间限制或建议放弃

4. **部分成功语义**
   - 需要明确定义什么算"成功"
   - 建议：至少50%班次成功才算部分成功，否则整体失败

5. **状态一致性**
   - 部分成功状态下的数据一致性保证
   - 建议：事务化处理或补偿机制
